# FTP Backup Tool - Инструмент резервного копирования на FTP

## 📋 Описание проекта

Инструмент для автоматического резервного копирования существующих многотомных архивов на FTP сервер с полной обработкой ошибок, логированием и очисткой старых копий.

## 🚀 Основные возможности

- 🔄 **Интеллектуальная синхронизация** - проверка наличия файлов на FTP перед копированием
- 🛡️ **Надежная обработка ошибок** - продолжение работы при ошибках копирования
- 📦 **Поддержка многотомных архивов** - работа с файлами формата `name_date_hash.tar.N`
- 🧹 **Автоматическая очистка** - удаление старых бэкапов (локальных и на FTP)
- 📝 **Ротация логов** - автоматическое архивирование и ограничение размера лог-файлов
- ⚙️ **Гибкая конфигурация** - настройка через INI-файл
- 🕐 **Cron-совместимость** - готовность к автоматическому запуску по расписанию

## 🎯 Принцип работы

### Алгоритм работы скрипта:

1. **Поиск архивов** - скрипт ищет многотомные архивы в указанной директории
2. **Проверка FTP** - для каждого набора проверяется наличие на FTP сервере
3. **Интеллектуальное копирование**:
   - Если набор полностью отсутствует на FTP - загружаются все тома
   - Если набор частично присутствует - загружаются только отсутствующие тома
   - Если набор полностью присутствует - пропускается
4. **Проверка целостности** - сравнение размеров локальных и удаленных файлов
5. **Очистка** - удаление старых копий на FTP и локально (только успешных)

### Формат архивов:

Скрипт поддерживает архивы в формате:
- `имя_дата_описание_хеш.tar` (первый том)
- `имя_дата_описание_хеш.tar.1` (второй том)
- `имя_дата_описание_хеш.tar.2` (третий том) и т.д.

## 🔧 Управление логами

### Автоматическая ротация:
- Лог файл ограничен 10 МБ
- При превышении размера файл архивируется в `.gz`
- Хранится максимум 10 архивных лог-файлов
- Старые логи автоматически удаляются

### Ручное управление:
```bash
# Просмотр логов
tail -f log/backup.log

# Проверка статуса
python check_status.py

# Очистка старых логов
python cleanup_logs.py
```

## 🚨 Коды возврата

- `0` - Успешное выполнение
- `1` - Выполнение с ошибками (некоторые наборы не загружены)
- `2` - Критическая ошибка (скрипт не смог выполнить основную работу)
- `130` - Прерывание пользователем (Ctrl+C)

## 🛠️ Устранение неполадок

### Распространенные проблемы:

1. **Ошибка подключения к FTP**:
   - Проверьте параметры подключения в `config.ini`
   - Убедитесь, что сервер доступен
   - Проверьте логин и пароль

2. **Файлы не копируются**:
   - Проверьте права на запись в удаленную директорию
   - Убедитесь, что есть место на FTP сервере
   - Проверьте файл `.ftpignore`

3. **Ошибки при проверке целостности**:
   - Убедитесь, что FTP сервер поддерживает команду `SIZE`
   - Проверьте сетевые подключения

4. **Логи не ротируются**:
   - Проверьте права на запись в директорию `log/`
   - Убедитесь, что есть место на диске

## 🔒 Безопасность

### Рекомендации по безопасности:

1. Храните `config.ini` в безопасном месте (не добавляйте в публичные репозитории)
2. Используйте сильные пароли для FTP
3. Ограничьте доступ к директории с логами
4. Регулярно обновляйте скрипт
5. Мониторьте логи на предмет подозрительной активности

## 📞 Поддержка

При возникновении проблем:

1. Проверьте логи в `log/backup.log`
2. Используйте `python check_status.py` для диагностики
3. Убедитесь, что конфигурация корректна
4. Проверьте доступность FTP сервера
